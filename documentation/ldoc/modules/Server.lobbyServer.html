reply (payload, ip, pn)
Reply to the client sending a command (Semantically convenient helper). The reply function, after a message is received from a connected player, is used to bounce a message back to the player.

Parameters:

  • payload
  • ip
  • pn
receiver ()
Receives incoming packets. This receiver function is the loop at the heart of the Lobby server. This function is responsible for managing all connections of players searching for or creating games. Players receive information on the available games and their connection information from this process.
